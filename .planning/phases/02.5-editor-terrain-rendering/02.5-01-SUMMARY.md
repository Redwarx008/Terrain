## Files Created/Modified
- `Terrain.Editor/Rendering/EditorTerrainEntity.cs` - Data class for single terrain entity with heightmap, LOD data, and GPU resources
- `Terrain.Editor/Rendering/EditorTerrainQuadTree.cs` - Simplified quadtree for LOD selection without streaming

- `Terrain.Editor/Services/HeightmapLoader.cs` - Extended with LoadHeightData, GenerateMinMaxErrorMaps, CreateHeightmapTexture methods

## Decisions Made
- Single Texture2D per terrain entity instead of Texture2DArray (simplifies memory management for editor)
- Generate MinMaxErrorMap at runtime from PNG (avoid preprocessing step for editor workflow)
- Reuse terrain shaders from Terrain/Effects/ (TerrainBuildLodLookup, TerrainBuildLodMap, TerrainBuildNeighborMask)
- SliceIndex always 0 in StreamInfo since single texture contains entire heightmap
