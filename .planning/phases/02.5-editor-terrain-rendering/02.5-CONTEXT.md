# Phase 2.5: Editor Terrain Rendering - Context

**Gathered:** 2026-04-01
**Status:** Ready for planning
**Source:** User discussion

<domain>
## Phase Boundary

Create simplified terrain rendering for editor with:
- Direct PNG heightmap loading (no preprocessing)
- Single texture per terrain entity (no streaming)
- Automatic grid splitting for heightmaps > 16k×16k
- Shared global LOD map for seamless chunk boundaries

**Key Constraint:** Do NOT modify `Terrain/` core library - Editor owns its rendering.

</domain>

<decisions>
## Implementation Decisions

### Architecture
- Editor implements independent terrain rendering (no dependency on core TerrainComponent/TerrainProcessor)
- Single Texture2D per terrain entity (no Texture2DArray, no streaming)
- Reuse or adapt shaders from `Terrain/Effects/` (they're just SDSL)

### Heightmap Loading
- Load PNG directly using ImageSharp
- Generate MinMaxErrorMap at runtime (copy logic from TerrainPreProcessor)
- No .terrain file format dependency for editor

### Terrain Splitting
- Split heightmaps > 16k×16k into 2×2, 3×3, etc. grid
- Each chunk is a separate EditorTerrainEntity
- 1 sample overlap on shared edges

### LOD System
- All entities share the same global LOD map
- Existing `TerrainBuildNeighborMask.sdsl` queries by coordinate (works across entities)
- Existing `ApplyCrackSnap()` handles cross-entity boundaries automatically
- **No enhanced crack snapping algorithm needed** - existing mechanism is sufficient

### Brush Editing
- Modifications near edges automatically propagate to adjacent chunks
- HeightEditor checks if brush affects multiple chunks
- GPU sync after modifications

### Claude's Discretion
- Shader reuse strategy (import vs. copy)
- Exact class structure and interfaces
- GPU resource management details

</decisions>

### New Files to Create

|------|---------|

| `Terrain.Editor/Rendering/EditorTerrainEntity.cs` | Single terrain entity (heightmap texture + LOD) |
| `Terrain.Editor/Rendering/EditorTerrainQuadTree.cs` | Simplified quadtree (no streaming) |
| `Terrain.Editor/Rendering/EditorTerrainRenderFeature.cs` | Stride render feature for editor terrain |
| `Terrain.Editor/Services/HeightmapLoader.cs` | Load PNG heightmaps, generate MinMaxErrorMaps |
| `Terrain.Editor/Services/TerrainSplitter.cs` | Split large heightmaps into chunks |
| `Terrain.Editor/Services/SplitTerrainConfig.cs` | Chunk configuration and coordinate system |

### Memory Calculation
- 16k×16k R16 = 512 MB GPU memory per terrain
- 32k×32k split into 2×2 grid = 4 × 512 MB = 2 GB total

### Splitting Algorithm
```
32k×32k heightmap → 2×2 grid
- Chunk (0,0): 0-16383 samples, world offset (0, 0, 0)
- Chunk (1,0): 16383-32767 samples, world offset (16383, 0, 0)
- Chunk (0,1): 0-16383 X, 16383-32767 Z, world offset (0, 0, 16383)
- Chunk (1,1): 16383-32767 samples, world offset (16383, 0, 16383)
```

</specifics>
