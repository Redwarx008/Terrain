**Added automatic heightmap splitting for large terrains with SplitTerrainConfig, TerrainSplitter, and multi-entity TerrainManager support.**

## Accomplishments
- SplitTerrainConfig computes chunk layout with 1-sample overlap on shared edges
- TerrainSplitter automatically splits heightmaps > 16k into grid chunks

- `Terrain.Editor/Services/TerrainSplitter.cs` - Large heightmap splitting into EditorTerrainEntity instances
- `Terrain.Editor/Services/TerrainManager.cs` - Multi-entity terrain management with List<EditorTerrainEntity>

- `Terrain.Editor/UI/Panels/SceneViewPanel.cs` - Updated to use HasTerrainLoaded and new TerrainLoadedEventArgs

## Decisions Made
- MaxChunkSize set to 16384 (16k) to limit GPU memory to ~512MB per chunk
- 1 sample overlap on shared edges ensures seamless editing across chunk boundaries
- Global height cache remains in TerrainManager for efficient raycast queries across all chunks
- World offset per entity enables correct height queries for split terrains
