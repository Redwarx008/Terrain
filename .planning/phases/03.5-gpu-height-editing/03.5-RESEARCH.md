# Phase 03.5: GPU Height Editing - Research

**Researched:** 2026-04-04
**Domain:** GPU render-to-texture height editing, SubViewport pattern, fragment shader brushing
**Confidence:** HIGH

## Summary

This research provides a **detailed technical analysis** of the Godot heightmap plugin's GPU height editing pattern, with exact code snippets for implementing the same approach in Stride.

**Primary recommendation:** Use the **SubViewport + 2-Sprite pattern** from the Godot heightmap plugin:
1. Create a viewport sized for the brush (with rotation margin)
2. Add a background sprite with `no_blend` shader to copy source region
3. Add a brush sprite with editing shader to apply the operation
4. Render once, then copy result back to source texture

**Why this pattern:**
- Proven in production for years (Zylann's godot_heightmap_plugin)
- Fragment shaders are simpler than compute shaders
- No UAV (Unordered Access View) complexity
- Works with Stride's existing RenderTarget infrastructure
- Delayed GPU readback ensures synchronization

## Godot HeightMap Plugin: Exact Rendering Pattern

**Source Analysis:** https://github.com/Zylann/godot_heightmap_plugin

### 1. SubViewport Setup (from painter.gd)

```gdscript
# From painter.gd _init() - Lines 45-74

func _init():
    _viewport = SubViewport.new()
    _viewport.size = Vector2(_brush_size, _brush_size)
    _viewport.render_target_update_mode = SubViewport.UPDATE_ONCE
    _viewport.render_target_clear_mode = SubViewport.CLEAR_MODE_ONCE
    # Require 4 components (RGBA)
    _viewport.transparent_bg = true

    # There is no "blend_disabled" option on standard CanvasItemMaterial...
    _no_blend_material = ShaderMaterial.new()
    _no_blend_material.shader = HT_NoBlendShader

    # Background sprite - copies source region unmodified
    _viewport_bg_sprite = Sprite2D.new()
    _viewport_bg_sprite.centered = false        # Critical: position is top-left
    _viewport_bg_sprite.material = _no_blend_material
    _viewport.add_child(_viewport_bg_sprite)

    # Brush sprite - applies editing operation
    _viewport_brush_sprite = Sprite2D.new()
    _viewport_brush_sprite.centered = true      # Critical: position is center
    _viewport_brush_sprite.material = _brush_material
    _viewport_brush_sprite.position = _viewport.size / 2.0
    _viewport.add_child(_viewport_brush_sprite)

    add_child(_viewport)
```

**Key Points:**
- `UPDATE_ONCE`: Manual control - only renders when we trigger it
- `CLEAR_MODE_ONCE`: Clears once at start of render
- `transparent_bg = true`: Needed for RGBA output
- Background sprite: `centered = false` (top-left positioning)
- Brush sprite: `centered = true` (center positioning)

The brush sprite uses a custom shader material with the brush shape texture:

**Critical:** The brush sprite's texture is the **brush shape mask** (a grayscale image defining falloff), NOT the heightmap.

### 5. Paint Input Sequence (Exact Order of Operations)

```gdscript
# From painter.gd - Lines 197-232

func paint_input(center_pos: Vector2):
    # 1. Calculate viewport size (fit rotated brush)
    var vp_size := _get_size_fit_for_rotation(Vector2(_brush_size, _brush_size))
    if _viewport.size != vp_size:
        # Lazy resize - don't lag the slider
        _viewport.size = vp_size
        _viewport_brush_sprite.position = _viewport.size / 2.0

    # 2. Calculate brush position (floor for integer coordinates)
    var brush_pos := (center_pos - _viewport.size * 0.5).round()

    # 3. Configure viewport for single render
    _viewport.render_target_update_mode = SubViewport.UPDATE_ONCE
    _viewport.render_target_clear_mode = SubViewport.CLEAR_MODE_ONCE

    # 4. Position background sprite to show correct source region
    _viewport_bg_sprite.position = -brush_pos

    # 5. Store position for later use
    _brush_position = brush_pos
    _cmd_paint = true

    # 6. Scale brush sprite to desired size
    _viewport_brush_sprite.scale = \
        _brush_scale * Vector2(_brush_size, _brush_size) \
        / Vector2(_viewport_brush_sprite.texture.get_size())

    # 7. Calculate source rect for shader (normalized)
    var rect := Color()
    rect.r = brush_pos.x / _texture.get_width()
    rect.g = brush_pos.y / _texture.get_height()
    rect.b = float(_viewport.size.x) / float(_texture.get_width())
    rect.a = float(_viewport.size.y) / float(_texture.get_height())

    # 8. Set shader parameters
    _brush_material.set_shader_parameter(SHADER_PARAM_SRC_RECT, rect)
    _brush_material.set_shader_parameter(SHADER_PARAM_OPACITY, _brush_opacity)
```

**Key Insights:**
- Background sprite position = `-brush_pos` (negative to offset viewport origin)
- Brush sprite position = `viewport.size / 2` (centered in viewport)
- Source rect is **normalized** (0-1 range relative to texture size)
- Brush sprite scale adjusts the brush shape to desired pixel size

### 6. Shaders: Height Editing Operations

#### Raise/Lower Shader (raise.gdshader)

```glsl
// From shaders/raise.gdshader - Entire file

shader_type canvas_item;
render_mode blend_disabled;

#include "res://addons/zylann.hterrain/shaders/include/heightmap.gdshaderinc"

uniform sampler2D u_src_texture;
uniform vec4 u_src_rect;
uniform float u_opacity = 1.0;
uniform float u_factor = 1.0;

vec2 get_src_uv(vec2 screen_uv) {
    vec2 uv = u_src_rect.xy + screen_uv * u_src_rect.zw;
    return uv;
}

void fragment() {
    float brush_value = u_factor * u_opacity * texture(TEXTURE, UV).r;
    float src_h = sample_heightmap(u_src_texture, get_src_uv(SCREEN_UV));
    float h = src_h + brush_value;
    COLOR = encode_height_to_viewport(h);
}
```

**Key Points:**
- `TEXTURE` = brush shape mask (from sprite)
- `UV` = brush texture coordinates
- `SCREEN_UV` = viewport coordinates (maps to source region)
- `u_src_rect` = normalized region being edited
- `u_factor` = height change (+ for raise, - for lower)

#### Smooth Shader (smooth.gdshader)

```glsl
// From shaders/smooth.gdshader - Entire file

shader_type canvas_item;
render_mode blend_disabled;

#include "res://addons/zylann.hterrain/shaders/include/heightmap.gdshaderinc"

uniform sampler2D u_src_texture;
uniform vec4 u_src_rect;
uniform float u_opacity = 1.0;
uniform float u_factor = 1.0;

vec2 get_src_uv(vec2 screen_uv) {
    vec2 uv = u_src_rect.xy + screen_uv * u_src_rect.zw;
    return uv;
}

float get_height(sampler2D heightmap, vec2 uv) {
    return sample_heightmap(heightmap, uv);
}

void fragment() {
    float brush_value = u_factor * u_opacity * texture(TEXTURE, UV).r;

    vec2 src_pixel_size = 1.0 / vec2(textureSize(u_src_texture, 0));
    vec2 src_uv = get_src_uv(SCREEN_UV);
    vec2 offset = src_pixel_size;

    // 5-point kernel: center + 4 neighbors
    float src_nx = get_height(u_src_texture, src_uv - vec2(offset.x, 0.0));
    float src_px = get_height(u_src_texture, src_uv + vec2(offset.x, 0.0));
    float src_ny = get_height(u_src_texture, src_uv - vec2(0.0, offset.y));
    float src_py = get_height(u_src_texture, src_uv + vec2(0.0, offset.y));
    float src_h = get_height(u_src_texture, src_uv);

    float dst_h = (src_h + src_nx + src_px + src_ny + src_py) * 0.2;
    float h = mix(src_h, dst_h, brush_value);
    COLOR = encode_height_to_viewport(h);
}
```

#### Level Shader (level.gdshader) - Average within brush region

```glsl
// From shaders/level.gdshader - Entire file

shader_type canvas_item;
render_mode blend_disabled;

#include "res://addons/zylann.hterrain/shaders/include/heightmap.gdshaderinc"

uniform sampler2D u_src_texture;
uniform vec4 u_src_rect;
uniform float u_opacity = 1.0;
uniform float u_factor = 1.0;

vec2 get_src_uv(vec2 screen_uv) {
    vec2 uv = u_src_rect.xy + screen_uv * u_src_rect.zw;
    return uv;
}

float get_height(sampler2D heightmap, vec2 uv) {
    return sample_heightmap(heightmap, uv);
}

void fragment() {
    float brush_value = u_factor * u_opacity * texture(TEXTURE, UV).r;

    // Sample 5x5 grid to approximate average
    float dst_h = 0.0;
    vec2 uv_min = vec2(u_src_rect.xy);
    vec2 uv_max = vec2(u_src_rect.xy + u_src_rect.zw);
    for (int i = 0; i < 5; ++i) {
        for (int j = 0; j < 5; ++j) {
            float x = mix(uv_min.x, uv_max.x, float(i) / 4.0);
            float y = mix(uv_min.y, uv_max.y, float(j) / 4.0);
            float h = get_height(u_src_texture, vec2(x, y));
            dst_h += h;
        }
    }
    dst_h /= (5.0 * 5.0);

    float src_h = get_height(u_src_texture, get_src_uv(SCREEN_UV));
    float h = mix(src_h, dst_h, brush_value);
    COLOR = encode_height_to_viewport(h);
}
```

#### Erode Shader (erode.gdshader) - Morphological erosion

```glsl
// From shaders/erode.gdshader - Entire file

shader_type canvas_item;
render_mode blend_disabled;

#include "res://addons/zylann.hterrain/shaders/include/heightmap.gdshaderinc"

uniform sampler2D u_src_texture;
uniform vec4 u_src_rect;
uniform float u_opacity = 1.0;
uniform vec4 u_color = vec4(1.0);

vec2 get_src_uv(vec2 screen_uv) {
    vec2 uv = u_src_rect.xy + screen_uv * u_src_rect.zw;
    return uv;
}

float get_height(sampler2D heightmap, vec2 uv) {
    return sample_heightmap(heightmap, uv);
}

float erode(sampler2D heightmap, vec2 uv, vec2 pixel_size, float weight) {
    float r = 3.0;

    // Divide so the shader stays neighbor dependent 1 pixel across.
    // For this to work, filtering must be enabled.
    vec2 eps = pixel_size / (0.99 * r);

    float h = get_height(heightmap, uv);
    float eh = h;

    // Morphology with circular structuring element
    for (float y = -r; y <= r; ++y) {
        for (float x = -r; x <= r; ++x) {
            vec2 p = vec2(x, y);
            float nh = get_height(heightmap, uv + p * eps);
            float s = max(length(p) - r, 0);
            eh = min(eh, nh + s);
        }
    }

    eh = mix(h, eh, weight);
    return eh;
}

void fragment() {
    float brush_value = u_opacity * texture(TEXTURE, UV).r;
    vec2 src_pixel_size = 1.0 / vec2(textureSize(u_src_texture, 0));
    float ph = erode(u_src_texture, get_src_uv(SCREEN_UV), src_pixel_size, brush_value);
    COLOR = encode_height_to_viewport(ph);
}
```

### 7. Heightmap Include File (heightmap.gdshaderinc)

This is the critical encoding/decoding logic:

```glsl
// From shaders/include/heightmap.gdshaderinc - Entire file

// Use functions from this file everywhere a heightmap is used,
// so it is easy to track and change the format

float sample_heightmap(sampler2D spl, vec2 pos) {
    // RF format - height in red channel
    return texture(spl, pos).r;
}

vec4 encode_height_to_viewport(float h) {
    // Encode regular floats into an assumed RGBA8 output color.
    // This is used because Godot 4.0 doesn't support RF viewports,
    // and the irony is, even if float viewports get supported, it's likely it will end up RGBAF,
    // which is wasting bandwidth because we are only interested in R...
    uint u = floatBitsToUint(h);
    return vec4(
        float((u >> 0u) & 255u),
        float((u >> 8u) & 255u),
        float((u >> 16u) & 255u),
        float((u >> 24u) & 255u)
    ) / vec4(255.0);
}

float decode_height_from_viewport(vec4 c) {
    uint u = uint(c.r * 255.0)
        | (uint(c.g * 255.0) << 8u)
        | (uint(c.b * 255.0) << 16u)
        | (uint(c.a * 255.0) << 24u);
    return uintBitsToFloat(u);
}

float sample_height_from_viewport(sampler2D screen, vec2 uv) {
    ivec2 ts = textureSize(screen, 0);
    vec2 norm_to_px = vec2(ts);

    // Convert to pixels and apply a small offset so we interpolate from pixel centers
    vec2 uv_px_f = uv * norm_to_px - vec2(0.5);

    ivec2 uv_px = ivec2(uv_px_f);

    // Get interpolation pixel positions
    ivec2 p00 = uv_px;
    ivec2 p10 = uv_px + ivec2(1, 0);
    ivec2 p01 = uv_px + ivec2(0, 1);
    ivec2 p11 = uv_px + ivec2(1, 1);

    // Get pixels
    vec4 c00 = texelFetch(screen, p00, 0);
    vec4 c10 = texelFetch(screen, p10, 0);
    vec4 c01 = texelFetch(screen, p01, 0);
    vec4 c11 = texelFetch(screen, p11, 0);

    // Decode heights
    float h00 = decode_height_from_viewport(c00);
    float h10 = decode_height_from_viewport(c10);
    float h01 = decode_height_from_viewport(c01);
    float h11 = decode_height_from_viewport(c11);

    // Linear filter
    vec2 f = fract(uv_px_f);
    float h = mix(mix(h00, h10, f.x), mix(h01, h11, f.x), f.y);

    return h;
}
```

**Stride Simplification:** Since Stride supports R16_Float/R32_Float render targets, encoding/decoding becomes:

```hlsl
// Stride version - no bit-packing needed
float SampleHeightmap(Texture2D tex, SamplerState sampler, float2 uv) {
    return tex.SampleLevel(sampler, uv, 0).r;
}

float4 EncodeHeightToViewport(float h) {
    return float4(h, 0, 0, 1);  // Direct output to R16_Float render target
}

float DecodeHeightFromViewport(float4 c) {
    return c.r;  // Direct read from R16_Float texture
}
```

### 8. Copy Result Back to Heightmap (Delayed Readback)

The critical synchronization pattern - render happens in one frame, readback in the next:

```gdscript
# From painter.gd _process() - Lines 245-293

func _process(delta: float):
    # Frame N+1: Read back viewport result (GPU work is complete)
    if _pending_paint_render:
        _pending_paint_render = false

        # Get viewport texture as image
        var viewport_image := _viewport.get_texture().get_image()

        # Handle format conversion (Godot 4.0 workaround)
        if _image.get_format() == Image.FORMAT_RF:
            # Reinterpret RGBA8 as RF
            assert(viewport_image.get_format() == Image.FORMAT_RGBA8)
            viewport_image = Image.create_from_data(
                viewport_image.get_width(), viewport_image.get_height(), false, Image.FORMAT_RF,
                viewport_image.get_data())
        elif _image.get_format() == Image.FORMAT_RH:
            # Reinterpret RGBA8 as RF then convert to RH
            assert(viewport_image.get_format() == Image.FORMAT_RGBA8)
            viewport_image = Image.create_from_data(
                viewport_image.get_width(), viewport_image.get_height(), false, Image.FORMAT_RF,
                viewport_image.get_data())
            viewport_image.convert(Image.FORMAT_RH)
        else:
            viewport_image.convert(_image.get_format())

        # Calculate source/destination regions
        var brush_pos := _last_brush_position

        var dst_x : int = clamp(brush_pos.x, 0, _texture.get_width())
        var dst_y : int = clamp(brush_pos.y, 0, _texture.get_height())

        var src_x : int = maxf(-brush_pos.x, 0)
        var src_y : int = maxf(-brush_pos.y, 0)
        var src_w : int = minf(maxf(_viewport.size.x - src_x, 0), _texture.get_width() - dst_x)
        var src_h : int = minf(maxf(_viewport.size.y - src_y, 0), _texture.get_height() - dst_y)

        # Apply to source texture
        if src_w != 0 and src_h != 0:
            _mark_modified_chunks(dst_x, dst_y, src_w, src_h)
            HT_Util.update_texture_partial(_texture, viewport_image,
                Rect2i(src_x, src_y, src_w, src_h), Vector2i(dst_x, dst_y))
            texture_region_changed.emit(Rect2(dst_x, dst_y, src_w, src_h))

    # Frame N: Input was handled, trigger render for next frame
    if _cmd_paint:
        _pending_paint_render = true
        _last_brush_position = _brush_position
        _cmd_paint = false
```

**Key Points:**
- `_cmd_paint` triggers render in current frame
- `_pending_paint_render` causes readback in next frame
- This 2-frame delay ensures GPU has finished rendering
- `update_texture_partial` copies the viewport region to the source texture

### 10. Viewport Size Calculation for Rotation

```gdscript
# From painter.gd - Lines 192-195

static func _get_size_fit_for_rotation(src_size: Vector2) -> Vector2i:
    # Diagonal of the brush rectangle fits any rotation
    var d = int(ceilf(src_size.length()))
    return Vector2i(d, d)
```

This ensures a rotated brush sprite doesn't get clipped at the viewport edges.

### Anti-Patterns to Avoid
- **Using CPU HeightDataCache as source of truth with GPU editing:** Creates sync complexity. GPU texture becomes authoritative during editing, CPU cache updated via readback only when needed (undo/save).
- **Immediate GPU readback after render:** GPU may not have finished. Use delayed readback pattern (render in frame N, readback in frame N+2).
- **Full heightmap undo snapshots:** Memory explosion. Use 64x64 chunk-based undo.
- **Creating new render targets per frame:** Allocation churn. Reuse viewport and resize only when brush size changes significantly.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Render-to-texture setup | Custom FBO management | Stride RenderTarget + Sprite approach | Proven pattern, simpler |
| Brush falloff calculation | Procedural in shader | Pre-made brush texture (EXR/PNG) | Better quality, artistic control |
| Undo storage | Full heightmap copy | 64x64 chunk-based storage | Memory efficient |
| Height encoding | Bit-packing to RGBA8 | R16_Float or R32_Float render target | Stride supports float formats |

**Key insight:** The Godot heightmap plugin has solved these problems in production for years. Adapt its proven patterns rather than inventing new approaches.

## Common Pitfalls

### Pitfall 1: Missing blend_disabled in Shaders
**What goes wrong:** Blending enabled causes incorrect height values when rendering to viewport.
**Why it happens:** Default alpha blending interpolates colors instead of replacing.
**How to avoid:** All editing shaders must use `render_mode blend_disabled;` (Godot) or equivalent blend state in Stride.
**Warning signs:** Height values drift, visible artifacts at brush edges.

### Pitfall 2: UV Coordinate Mismatch
**What goes wrong:** Brush edits wrong region or shows seams.
**Why it happens:** Screen-space UV vs texture UV confusion in shader.
**How to avoid:** Follow Godot's UV transformation pattern: `u_src_rect.xy + screen_uv * u_src_rect.zw`.
**Warning signs:** Edits offset from cursor, partial updates.

### Pitfall 3: GPU-CPU Sync Race Condition
**What goes wrong:** Readback returns stale or corrupted data.
**Why it happens:** Reading GPU results before rendering completes.
**How to avoid:** Use 2-frame delay pattern: render frame N, readback frame N+2.
**Warning signs:** Intermittent corruption, undo doesn't match actual changes.

### Pitfall 4: Viewport Size Too Small for Rotation
**What goes wrong:** Rotated brush is clipped at edges.
**Why it happens:** Viewport size equals brush diameter, but rotation increases bounding box.
**How to avoid:** Viewport size = `ceil(length(brush_size, brush_size))` (diagonal).
**Warning signs:** Brush shape cut off when rotated 45 degrees.

### Pitfall 5: Undo Chunk Boundary Issues
**What goes wrong:** Undo leaves seams at chunk boundaries.
**Why it happens:** Brush edits cross chunk boundaries but only some chunks stored.
**How to avoid:** Mark ALL chunks touched by brush region, not just center chunks.
**Warning signs:** Visible grid pattern after undo.

```csharp
public sealed class GpuHeightEditor : IDisposable
{
    private const int UndoChunkSize = 64;

    private readonly GraphicsDevice _graphicsDevice;
    private Texture? _renderTarget;
    private Sprite? _backgroundSprite;
    private Sprite? _brushSprite;
    private Texture? _sourceHeightmap;

    // Shader materials
    private readonly Material _noBlendMaterial;
    private readonly Material _raiseMaterial;
    private readonly Material _smoothMaterial;
    private readonly Material _flattenMaterial;

    public GpuHeightEditor(GraphicsDevice graphicsDevice)
    {
        _graphicsDevice = graphicsDevice;

        // Create materials from shaders
        _noBlendMaterial = CreateMaterial("HeightEditCopy");
        _raiseMaterial = CreateMaterial("HeightEditRaise");
        _smoothMaterial = CreateMaterial("HeightEditSmooth");
        _flattenMaterial = CreateMaterial("HeightEditFlatten");
    }

    public void InitializeRenderTarget(int brushSize)
    {
        // Size viewport to fit rotated brush (diagonal)
        int viewportSize = (int)Math.Ceiling(Math.Sqrt(2) * brushSize);

        _renderTarget = Texture.New2D(
            _graphicsDevice,
            viewportSize,
            viewportSize,
            1,
            PixelFormat.R16_Float,  // Stride supports float render targets directly
            TextureFlags.RenderTarget | TextureFlags.ShaderResource);
    }

    public void PaintRaise(Vector2 brushCenter, int brushSize, float strength, float opacity)
    {
        // Set shader parameters
        _raiseMaterial.Parameters.Set(HeightEditRaiseKeys.SourceTexture, _sourceHeightmap);
        _raiseMaterial.Parameters.Set(HeightEditRaiseKeys.SourceRect, CalculateSourceRect(brushCenter, brushSize));
        _raiseMaterial.Parameters.Set(HeightEditRaiseKeys.Opacity, opacity);
        _raiseMaterial.Parameters.Set(HeightEditRaiseKeys.Factor, strength);

        // Render to target
        RenderBrushToTarget(brushCenter, brushSize);

        // Schedule delayed readback
        ScheduleReadback(brushCenter, brushSize);
    }

    private void RenderBrushToTarget(Vector2 brushCenter, int brushSize)
    {
        using var drawContext = _graphicsDevice.CreateDrawContext();

        // Set render target
        drawContext.CommandList.SetRenderTarget(_renderTarget);
        drawContext.CommandList.Clear(_renderTarget, Color.Transparent);

        // Render background sprite (copy source region)
        _backgroundSprite.Position = CalculateBackgroundPosition(brushCenter);
        _backgroundSprite.Draw(drawContext);

        // Render brush sprite (apply edit)
        _brushSprite.Position = new Vector2(_renderTarget.Width, _renderTarget.Height) / 2f;
        _brushSprite.Draw(drawContext);
    }
}
```

### Stride SDSL Height Editing Shader

```hlsl
// HeightEditRaise.sdsl - Adapted from Godot raise.gdshader
namespace Terrain.Editor
{
    shader HeightEditRaise : ShaderBase
    {
        // Standard painter parameters (matching Godot convention)
        stage Texture2D SourceTexture;
        stage float4 SourceRect;  // xy = offset (normalized), zw = size (normalized)
        stage float Opacity = 1.0f;
        stage float Factor = 1.0f;

        // Brush mask texture (assigned to sprite's texture)
        // In Stride, the sprite's texture is available via Texture semantic

        float2 GetSourceUV(float2 screenUV)
        {
            return SourceRect.xy + screenUV * SourceRect.zw;
        }

        float SampleHeightmap(float2 uv)
        {
            return SourceTexture.SampleLevel(Sampler, uv, 0).r;
        }

        float4 EncodeHeightToViewport(float h)
        {
            // In Stride, we can use R16_Float directly - no bit-packing needed
            return float4(h, 0, 0, 1);
        }

        override void Fragment()
        {
            // Brush mask value from sprite texture
            float brushValue = Factor * Opacity * Texture.SampleLevel(Sampler, streams.TexCoord, 0).r;

            // Sample current height from source
            float srcH = SampleHeightmap(GetSourceUV(streams.ScreenPosition.xy));

            // Apply change
            float h = srcH + brushValue;

            // Output modified height
            streams.ColorTarget = EncodeHeightToViewport(h);
        }
    };
}
```

### Chunk-Based Undo Manager

```csharp
// HeightEditUndoManager.cs - Adapted from Godot's commit pattern
public sealed class HeightEditUndoManager
{
    private const int ChunkSize = 64;

    private readonly Dictionary<Vector2I, UndoChunk> _modifiedChunks = new();
    private readonly Image _sourceImage;

    public void MarkModifiedChunks(int brushX, int brushY, int brushWidth, int brushHeight)
    {
        int cminX = brushX / ChunkSize;
        int cminY = brushY / ChunkSize;
        int cmaxX = (brushX + brushWidth - 1) / ChunkSize + 1;
        int cmaxY = (brushY + brushHeight - 1) / ChunkSize + 1;

        for (int cy = cminY; cy < cmaxY; cy++)
        {
            for (int cx = cminX; cx < cmaxX; cx++)
            {
                var chunkPos = new Vector2I(cx, cy);
                if (!_modifiedChunks.ContainsKey(chunkPos))
                {
                    _modifiedChunks[chunkPos] = new UndoChunk
                    {
                        Position = chunkPos,
                        InitialData = CaptureChunkInitial(chunkPos)
                    };
                }
            }
        }
    }

    public UndoSnapshot Commit(Image finalImage)
    {
        var snapshot = new UndoSnapshot();

        foreach (var (pos, chunk) in _modifiedChunks)
        {
            chunk.FinalData = CaptureChunkFinal(pos, finalImage);
            snapshot.Chunks.Add(chunk);
        }

        _modifiedChunks.Clear();
        return snapshot;
    }

    private sealed class UndoChunk
    {
        public Vector2I Position;
        public byte[] InitialData;
        public byte[] FinalData;
    }
}
```

## Open Questions

1. **Undo/Redo Integration with GPU Editing**
   - What we know: Phase 4 (UNDO-01 to UNDO-05) is planned but not yet implemented. Current code has no undo system.
   - **Godot approach:** Chunk-based undo (64x64 pixels) - capture initial chunks before edit, final chunks from GPU readback on commit.
   - Recommendation: Implement chunk-based undo following Godot's proven pattern. Store 64x64 pixel chunks instead of full heightmap snapshots.

2. **Multi-Slice Editing Strategy**
   - What we know: `EditorTerrainEntity` supports up to 8 slices (`MaxBoundHeightSlices = 8`). Current CPU editing marks dirty slices and syncs each independently.
   - What's unclear: Should GPU editing use separate render targets per slice (simple but multiple renders) or unified staging buffer (complex but single render)?
   - Recommendation: One render target per slice for simplicity; slice count is typically low (1-4 for most terrains).

3. **CPU-GPU Coherency Model**
   - What we know: Current architecture uses CPU cache as source of truth, syncing to GPU.
   - **Godot approach:** CPU image is authoritative for undo. GPU edits flow back to CPU via readback after each stroke.
   - Recommendation: Follow Godot's pattern - maintain CPU cache for undo/save, sync from GPU after editing operations complete.

4. **Performance Threshold for GPU vs CPU**
   - What we know: CPU editing iterates O(pi * radius^2) pixels per frame; GPU render is constant overhead + parallel execution.
   - What's unclear: At what brush radius does GPU become faster than CPU?
   - Recommendation: Benchmark both approaches. Tentative threshold: radius > 20px use GPU, otherwise CPU.
