# 决策记录索引

本目录存储架构决策记录（Architecture Decision Records, ADR）。

---

## 什么是 ADR？

架构决策记录用于记录重要的技术决策，包括：
- 决策背景
- 考虑的选项
- 最终决定
- 权衡分析

---

## 如何创建新的决策记录

1. 复制 `TEMPLATE.md` 到 `adr-[number]-[short-name].md`
2. 填写各个部分
3. 在本文件底部添加索引条目
4. 编号规则：从 011 开始（001-010 预留给早期会话日志期间的决策）

### ⚠️ 重要原则

**ADR 必须基于当前代码状态编写，不要从旧日志中推测。** 如果对当前实现不确定，先用代码验证再写 ADR。

### 何时创建 ADR？

- 同一决策在 3 个以上会话中被引用
- 做出了不可逆的技术选择
- 影响多个子系统的设计决定
