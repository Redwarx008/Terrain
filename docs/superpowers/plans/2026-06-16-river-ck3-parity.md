# River CK3 Parity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the Editor river `bottom -> refraction -> surface` chain so the riverbed, bank fade, and refraction behavior follow CK3 semantics instead of the current world-UV and scene-seed shortcut path.

**Architecture:** The work is split into three technical layers. First, text tests are rewritten so they stop locking in the known-wrong world-space bottom UV and old alpha path. Second, river settings are pushed from `RiverComponent` into `RiverRenderObject` and then into shader parameters so bottom/surface no longer depend on hard-coded shader defaults. Third, `RiverBottom.sdsl`, `RiverRenderFeature`, and `RiverSurface.sdsl` are updated together so bottom uses CK3-style advanced UV/depth/bank fade semantics, the refraction seed is separated from the working bottom buffer, and the surface consumes the corrected buffer with explicit bank fade semantics. RenderDoc verification is mandatory before the work is considered done.

**Tech Stack:** C#/.NET, Stride render features and DynamicEffect shaders, SDSL shader assets, RenderDoc frame captures, existing `Terrain.Editor.Tests` console text-test harness.

- Modify: `Terrain.Editor.Tests/RiverShaderTextTests.cs`  
  Replaces stale assertions that currently lock the implementation to the known-wrong world-space bottom UV, old bottom alpha, and all-linear texture import assumptions.
- Modify: `Terrain.Editor/Rendering/River/RiverRenderObject.cs`  
  Stores a runtime snapshot of river rendering settings so shader parameters can be bound per render object.
- Modify: `Terrain.Editor/Rendering/River/RiverProcessor.cs`  
  Pushes `RiverComponent.Settings` into each `RiverRenderObject` every frame instead of only syncing mesh data.
- Modify: `Terrain.Editor/Rendering/River/RiverRenderResources.cs`  
  Adds a distinct half-resolution scene-seed texture so scene seed and working refraction color are no longer the same resource.
- Modify: `Terrain.Editor/Rendering/River/RiverRenderFeature.cs`

```c
float CalcBottomDepth(float2 tangentUv, float4 bottomNormalSample)
{
    float shoreAmount = 1.0f + _BankAmount;
    float centerOffset = (shoreAmount - 1.0f) * 0.5f;
    float crossSection = clamp(tangentUv.y * shoreAmount - centerOffset, 0.0f, 1.0f);
    float depth = RiverDepthFromCrossSection(crossSection, _Depth, _DepthWidthPower);
    float sampledDepth = 1.0f - bottomNormalSample.b;
    return clamp(depth * sampledDepth, 0.001f, 10.0f);
}

void CalculateParallaxOffsetSteep(float3 tangentSpaceToCameraDir, float3 worldSpaceToCameraDir, float2 tangentUv, float worldWidth, out float2 tangentSpaceOffset, out float2 worldSpaceOffset)
{
    int minNumLayers = 2;
    int maxNumLayers = max(_ParallaxIterations, 2);
    float numLayers = lerp((float)maxNumLayers, (float)minNumLayers, saturate(worldSpaceToCameraDir.y));
    float layerDepth = _Depth / numLayers;
    float currentDepth = 0.0f;

    float4 step = float4(0.0f);
    float4 offset = float4(0.0f);
    float4 normalSample = BottomNormalTexture.Sample(BottomTextureSampler, tangentUv);
    float depth = CalcBottomDepth(tangentUv, normalSample);

    step.xy = (((-tangentSpaceToCameraDir.xy * _Depth) / max(tangentSpaceToCameraDir.z, 0.0001f)) / numLayers);
    step.zw = (((-worldSpaceToCameraDir.xz * _Depth) / max(worldSpaceToCameraDir.y, 0.0001f)) / numLayers);
    step.x *= _TextureUvScale;
    step.zw *= worldWidth;

    for (int i = 0; i < maxNumLayers; ++i)
    {
        if (depth <= currentDepth)
        {
            break;
        }

        currentDepth += layerDepth;
        offset += step;
        normalSample = BottomNormalTexture.Sample(BottomTextureSampler, tangentUv + offset.xy);
        depth = CalcBottomDepth(tangentUv + offset.xy, normalSample);
    }

    float4 previousNormalSample = BottomNormalTexture.Sample(BottomTextureSampler, tangentUv + offset.xy - step.xy);
    float prevDepth = CalcBottomDepth(tangentUv + offset.xy - step.xy, previousNormalSample) - currentDepth + layerDepth;
    float nextDepth = depth - currentDepth;
    float weight = nextDepth / max(nextDepth - prevDepth, 0.0001f);
    offset -= step * weight;

    tangentSpaceOffset = offset.xy;
    worldSpaceOffset = offset.zw;
}

void CalcParallaxedBottomUvs(float2 riverUv, float3 positionWS, float3x3 tbn, float worldWidth, float3 viewDir, out float2 worldUv, out float2 tangentUv)
{
    float2 scaledRiverUv = float2(riverUv.x * _TextureUvScale, riverUv.y);
    float3x3 invTbn = transpose(tbn);
    float3 tangentSpaceToCameraDir = mul(viewDir, invTbn);

    float2 tangentSpaceOffset;
    float2 worldSpaceOffset;
    CalculateParallaxOffsetSteep(tangentSpaceToCameraDir, viewDir, scaledRiverUv, worldWidth, tangentSpaceOffset, worldSpaceOffset);

    worldUv = positionWS.xz + worldSpaceOffset;
    tangentUv = scaledRiverUv + tangentSpaceOffset;
}

float3 ComputeDepthShapedNormal(float2 tangentUv, float underOceanFade, float3x3 tbn, float3 normalSample)
{
    float sampleWidth = 0.1f;
    float2 depthSampleOffset = float2(0.0f, sampleWidth * 0.5f);
    float4 beforeSample = BottomNormalTexture.Sample(BottomTextureSampler, tangentUv - depthSampleOffset);
    float4 afterSample = BottomNormalTexture.Sample(BottomTextureSampler, tangentUv + depthSampleOffset);
    float depthBefore = CalcBottomDepth(tangentUv - depthSampleOffset, beforeSample);
    float depthAfter = CalcBottomDepth(tangentUv + depthSampleOffset, afterSample);
    float slope = ((depthBefore - depthAfter) * underOceanFade) / sampleWidth;
    float angle = atan(slope);
    float3 parallaxNormal = float3(0.0f, -sin(angle), cos(angle));
    parallaxNormal.xy += normalSample.xy;
    return normalize(mul(parallaxNormal, tbn));
}
```

```markdown
## Confirmed Pattern: CK3 River Bottom Needs Tangent-UV Semantics

- CK3 riverbed textures are identical to the imported DDS assets in this repo, so a dark/black bank result is more likely to be a shader-semantic bug than an asset bug.
- If a bottom pass looks like a uniform dark ribbon, verify whether the shader is still sampling `worldPos.xz` directly. CK3 advanced rivers need tangent-UV plus parallax semantics before bank color will appear.

## Confirmed Pattern: Separate Refraction Scene Seed From The Working Bottom Buffer

- A single texture serving as both scene seed and bottom/refraction working target makes RenderDoc analysis ambiguous and encourages accidental over-filtering.
- Keep a dedicated half-resolution scene-seed texture, then copy it into the working bottom buffer before the dual-source bottom pass.
```
